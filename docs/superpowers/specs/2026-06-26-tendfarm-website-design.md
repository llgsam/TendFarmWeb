# TendFarm 网站设计方案

**日期**：2026-06-26  
**状态**：已审核，待实现

---

## 1. 背景与目标

TendFarm 是一个由真实健康数据驱动的放置建造农场（iOS App，基于 HealthKit / Apple Watch）。网站的核心目标是：

1. 通过 SEO 持续获取自然流量
2. 让访客理解 TendFarm 的产品定位与健康理念
3. 引导用户加入候补名单（TestFlight / 上线通知）

当前 App 尚未上架，**不放 App Store 下载链接**，以候补名单邮件收集为唯一转化动作。

---

## 2. 技术选型

| 项目 | 选择 | 理由 |
|---|---|---|
| 框架 | **Next.js（SSG 静态生成）** | SSG 输出纯 HTML，SEO 原生友好；支持组件化开发和未来交互扩展 |
| 语言 | **TypeScript** | 类型安全，长期维护成本低 |
| 样式 | **Tailwind CSS** | 快速迭代，与 Next.js 生态匹配 |
| 部署 | **Vercel** | 与 Next.js 无缝，自动 HTTPS + CDN |
| 邮件收集 | **Supabase（存储）+ Resend（发信）** | 数据自主可控，Resend 发确认邮件，后续可触达 |
| 国际化 | **next-intl** | 双语路由 `/zh` `/en`，SEO 友好的 `hreflang` |

## 3. 站点架构

### 3.1 路由结构

```
/                        → 语言检测 → 重定向到 /zh 或 /en
/zh/                     → 中文首页
/zh/philosophy           → 产品理念
/zh/lifestyle            → 健康生活方式内容枢纽
/zh/gameplay             → 玩法体验
/zh/tools                → 工具集（占坑，后续填充）
/zh/guides/              → 农场游戏攻略中心
/zh/guides/hay-day/      → Hay Day 攻略系列
/zh/guides/stardew-valley/
/zh/guides/animal-crossing/
/zh/guides/[game]/[slug] → 具体攻略文章

/en/                     → 英文镜像（相同结构）
```

### 3.2 页面职责

| 页面 | 核心目标 | SEO 方向 |

| 首页 | 品牌第一印象 + 候补收集 | 健康农场游戏、health farm game |

## 4. 各页面设计

### 4.1 首页

区块从上到下：

1. **Hero**
   - 中文主标题：「你的健康生活，在这里生长」
   - 英文主标题：「Your healthy life, growing here」
   - 副标题（中）：睡得好，作物更充盈。动得多，农场提前丰收。TendFarm 把你的真实节律变成一座会自动生长的农场。
   - 副标题（英）：Sleep well, harvest more. Move more, crops ripen early. TendFarm turns your real-life rhythm into a self-growing farm.
   - CTA：「加入候补名单 →」（跳转到邮件表单）
   - 背景：等距农场插图（SVG 矢量，复用 App asset）

2. **核心概念（3 列卡片）**
   - Activity → 阳光 → 作物提前成熟
   - Vitals / 睡眠 → 晨露 → 收成更充盈
   - 生活节律 → 转化率 → 农场更懂你

3. **玩法循环简介**
   - 一句话：「开田即生产，活动让作物提前成熟」
   - 简化流程图动效：田地 → 浇水 → 生长 → 收成 → 清仓 → 升级

4. **攻略内容 Teaser**
   - 「你玩这些游戏吗？」
   - 3 张攻略卡片预览（Hay Day / 星露谷 / 动物森友会）
   - → 进入攻略中心

5. **候补名单表单（二次触达）**
   - 标题：「第一批 TestFlight 内测资格」
   - Email 输入框 + 提交按钮
   - 提交后：显示感谢文案，不跳转

6. **Footer**
   - 语言切换 / 联系方式 / 隐私政策 / 版权

### 4.2 产品理念页（/philosophy）

内容结构：

1. **一句话定义**（全宽 Hero）
   > TendFarm 是一个健康生活显化农场：它把你的 Activity、睡眠、HRV 和长期生活节律转化为农场的生长速度、收成充盈、燃料效率和等级成长。

2. **设计原则 4 条**（交错排版）
   - 健康数据不直接给金币，而是影响效率
   - 普通用户无需规划，高级用户可以优化
   - 生活节律是生产转化率，不是装饰
   - 自动化是离线托管，手动操作是在线兜底

3. **Health Data → 农场 映射表**
   - Activity → 阳光 → 生长速度
   - Vitals → 晨露 → 收成充盈度
   - HRV → 燃料利用率
   - 生活节律 → 健康转化率（×1.00 ~ ×1.12）
   - 长期数据 → LifeExp → 农场等级

4. **CTA**：加入候补名单

### 4.3 健康生活方式页（/lifestyle）

定位：SEO 内容枢纽。所有 Section 在同一页面（不拆独立路由），页面权重集中；每个 Section 有独立锚点 `#hrv` 等，支持直链分享。

Section 列表：
- 什么是生活节律（circadian rhythm）
- HRV 是什么，为什么重要
- 睡眠质量如何影响白天状态
- 如何建立稳定的活动节律
- Apple Watch 健康数据怎么看

每个 Section：400-600 字 + 1 张插图 + 相关关键词自然植入。底部统一 CTA 到候补名单。

### 4.4 玩法体验页（/gameplay）

1. **核心循环可视化**
   - 动画流程图：田地状态机（待浇水 → 生长 → 成熟 → 收成 → 入仓 → 清仓）
   - 水系统：水池 → 自动浇水机 → 田地

2. **Health Data 影响演示**
   - 静态对比卡：「今天活动多」vs「今天久坐」的农场产出差异
   - 展示晨露、生活节律分档文案

3. **作物矩阵展示**
   - 5 种作物（晨光豆 / 小麦芽 / 棉花团 / 夜莓 / 能量草）卡片
   - 各自的周期、适合场景、燃料系数

4. **前 7 天节奏预告**
   - Day 1-7 各阶段体验简介

5. **CTA**：加入候补名单

## 5. SEO 策略

### 5.1 关键词矩阵

| 类型 | 中文 | 英文 |
|---|---|---|
| 品牌词 | TendFarm、健康农场游戏 | TendFarm, health farm game |
| 理念词 | 健康数据游戏化、步数游戏 | health data gamification, steps game |
| 健康词 | 生活节律、HRV 是什么、睡眠质量 | circadian rhythm, what is HRV |
| 游戏词（攻略） | Hay Day 攻略、星露谷攻略 | Hay Day guide, Stardew Valley guide |
| 工具词 | 生活节律测评（预留） | lifestyle assessment（预留）|

### 5.2 内链漏斗

```
攻略文章 → 理念页 / 玩法页 → 首页候补表单
```

每篇攻略文章末尾植入 TendFarm 钩子，引导到理念页或玩法页，玩法页和理念页底部指向候补表单。

### 5.3 技术 SEO
- 每页独立 `<title>` 和 `<meta description>`（Next.js Metadata API）
- `hreflang` 标记中英双语对应关系
- `sitemap.xml` 自动生成（next-sitemap）
- Open Graph 图片（每页独立，方便社交分享）
- 结构化数据：攻略文章用 `Article` schema，工具页用 `WebApplication` schema

## 6. 视觉风格

- **色调**：暖色暗场景底（深绿 `#1a2e1a` / 深棕 `#2d1f0e` / 琥珀 `#f0a832`），与 App 等距农场一脉相承
- **字体**（中）：思源宋体（正文）/ 思源黑体（UI 标签）
- **字体**（英）：Lora（正文）/ Inter（UI 标签）
- **插图**：SVG 矢量等距农场插图，复用 App asset_pack 素材
- **动效**：克制，CSS 动画仅用于 Hero 区和玩法页（作物生长进度条、水位变化）
- **攻略页**：以游戏截图 + 标注为主，轻量卡通风格标注框

---

## 7. 候补名单系统

- **前端**：Email 输入表单（Next.js Server Action 或 API Route）
- **后端存储**：Supabase 表（`waitlist` — email, locale, source_page, created_at）
- **去重**：同一 email 只记录一次，提示「你已在候补名单中」
- **确认邮件**：Resend 发送确认邮件（可选，MVP 可以不做）
- **来源追踪**：`source_page` 字段记录从哪个页面提交，便于分析哪个内容转化最好

## 9. 关键约束

- 不放 App Store 下载链接（App 尚未上架）
- 候补名单是唯一转化动作
- 工具集暂不实现具体工具，仅占坑

- 视觉延续 App 风格，不另起炉灶
